MiniClaw MiniApp for MiniPay — Build Prompt
Overview
Build a lightweight, mobile-first MiniClaw miniapp designed to run inside MiniPay (Celo's mobile wallet). The app is a standalone frontend project that connects to the SelfClaw API (https://selfclaw.ai) to provide the full MiniClaw AI assistant experience — creating, chatting with, configuring, and managing personal AI agents.

The miniapp should feel native to MiniPay: fast, minimal, and wallet-first.

Tech Stack
Frontend: React or vanilla HTML/JS — keep it lightweight
Styling: Light, clean, elegant. Think modern fintech / consumer app — soft colors, rounded corners, generous whitespace. Not the SelfClaw brutalist aesthetic — this should feel approachable and friendly.
No heavy frameworks — loads inside MiniPay's embedded browser; keep bundle size small
Use ethers.js v6 or viem for wallet interactions
Design Direction
Light and elegant — clean whites/light grays, subtle shadows, soft accent color
Simple and clear — no visual clutter, easy to scan, obvious actions
Mobile-native — touch-friendly, smooth transitions, bottom-anchored inputs
NOT the SelfClaw look — no brutalist borders, no monospace everywhere, no dark-first palette. This is its own product with its own lighter identity.
The designer has creative freedom on colors, typography, and layout — just keep it simple, readable, and polished.
Authentication
MiniPay provides window.ethereum with isMiniPay: true. Auth flow:

Detect MiniPay: Check window.ethereum.isMiniPay || window.ethereum.isMinipay or user-agent containing "MiniPay"
Request accounts: window.ethereum.request({ method: 'eth_requestAccounts' }) → get wallet address
Authenticate with SelfClaw API:
POST /api/auth/self/wallet/minipay-token → get a one-time token
POST /api/auth/self/wallet/minipay-connect with { address, token } → establishes session
Session: Use cookies (credentials: 'include' on all fetch calls). The wallet address IS the user identity — no email, no password.
SelfClaw API Reference
Base URL: https://selfclaw.ai Auth: Session cookie (set after MiniPay connect flow). All requests use credentials: 'include'.

Auth Endpoints
Method	Endpoint	Description
POST	/api/auth/self/wallet/minipay-token	Get a one-time auth token
POST	/api/auth/self/wallet/minipay-connect	Connect wallet. Body: { address, token }
GET	/api/auth/status	Check current session. Returns { loggedIn, user }
Agent Management
Method	Endpoint	Description
GET	/api/selfclaw/v1/hosted-agents	List user's agents
GET	/api/selfclaw/v1/hosted-agents/:id	Get single agent details
POST	/api/selfclaw/v1/hosted-agents	Create agent. Body: { name, emoji, description, interests, topicsToWatch, personaTemplate, humorStyle }
PATCH	/api/selfclaw/v1/hosted-agents/:id	Update agent (status, name, emoji)
DELETE	/api/selfclaw/v1/hosted-agents/:id	Delete an agent
GET	/api/selfclaw/v1/hosted-agents/templates	Get persona templates
GET	/api/selfclaw/v1/hosted-agents/skills	List available skills
POST	/api/selfclaw/v1/hosted-agents/:id/skills/:skillId/enable	Enable a skill
POST	/api/selfclaw/v1/hosted-agents/:id/skills/:skillId/disable	Disable a skill

Features (Full MiniClaw Parity)
The miniapp should support all of these — the specific UI/UX is up to the designer:

Wallet connect — one-tap connection using MiniPay's injected wallet
Agent list — see all your MiniClaws, their status, tap to chat
Create agent — pick a persona template, name it, set emoji, description, humor style, choose skills, add knowledge, deploy
Chat — real-time streaming conversation with your agent, conversation history
Memory management — browse, pin, edit, delete memories the agent has formed
Knowledge upload — add text or URLs to teach the agent specific topics (up to 20 entries), view and delete existing entries
Skill management — toggle skills on/off (Wallet Monitor, Price Watcher, Economics Tracker, News Radar, Research Assistant, Smart Advisor, Content Helper, Reputation Monitor)
Agent settings — edit name, emoji, description, interests, topics, humor style, model tier, social handles
Persona templates — 6 options: General Assistant, Research Analyst, DeFi Trader, Community Manager, Personal Coach, Support Agent
Soul document — view and edit the agent's evolving identity document
Task approvals — view pending autonomous tasks, approve or reject them
Telegram connection — connect/disconnect Telegram bot, configure settings
Delete agent — ability to remove an agent entirely
MiniPay-Specific Requirements
Mobile-only viewport, no desktop layout needed (max ~480px)
maximum-scale=1.0, user-scalable=no
Use 100dvh for full-height screens
All navigation stays within the miniapp (hash-based routing or view switching)
Keep initial JS bundle small — lazy-load non-critical screens
Graceful error/offline states
All API calls to https://selfclaw.ai with credentials: 'include'
