What & Why
The external miniapp at miniclawapp.replit.app consumes the GET /v1/hosted-agents list endpoint, but several fields it needs are missing from the response. The highest-value gap is runtimeStatus — without it, all agents always appear "active" and the animated thinking/running indicators never fire. Secondary gaps include currentActivity (agent card subtitle), pocScore, and celoBalance.

Done looks like
The GET /v1/hosted-agents list response includes runtimeStatus ("thinking" | "running" | "idle") per agent, derived from whether a task is currently being processed by the worker
The list response includes currentActivity per agent (same logic as the detail endpoint)
The list response includes pocScore per agent (joined from poc_scores table)
Quota headers (X-Quota-Tokens-Used, X-Quota-Tokens-Remaining, X-Quota-Reset) are returned on the poll result endpoint (GET /:id/chat/:messageId/result) — already partially there, verify and confirm
The miniclaw-api.html docs page is updated to document the new fields in the agent list response
No breaking changes to existing response shape (additive only)
