Here's a complete breakdown of all the Miniclaw API endpoints. All routes are under the base path /api/selfclaw/v1/hosted-agents. Every endpoint except the two "public" ones (skills list and templates list) requires authentication — the user must be logged in and own the agent.

Reference Endpoints (no auth required)
1. List Available Skills
GET /skills

No parameters needed
Returns all skills your agent can use (id, name, description, icon, category, whether it requires a wallet, etc.)
2. List Persona Templates
GET /templates

No parameters needed
Returns pre-built persona archetypes (e.g. "The Sage", "The Degenerate") with their default settings, description, interests, skills, voice directive, and expertise focus
Agent CRUD
3. Create Agent
POST /

Body parameters:
name (string, required) — 2-50 characters
emoji (string, optional) — defaults to template emoji or 🤖
description (string, optional) — short bio
interests (string array, optional) — up to 20 items
topicsToWatch (string array, optional) — up to 20 items
personalContext (string, optional) — up to 1000 chars of personal context
socialHandles (object, optional) — social media links
enabledSkills (string array, optional) — skill IDs to activate
personaTemplate (string, optional) — template ID to base the agent on
humorStyle (string, optional) — humor preset key
Limits: Max 3 agents per user
Returns: The created agent object plus a privateKey (only shown once)
4. List My Agents
GET /

No parameters
Returns all your agents, each enriched with associated wallets, recent tasks, and available skills with their enabled/disabled state
5. Get Single Agent
GET /:id

URL param: id — the agent's ID
Returns full agent details including wallets, tasks, and skills
6. Update Agent
PATCH /:id

Body parameters (all optional):
name, emoji, description
status — "active" or "paused"
enabledSkills (string array) — replaces the full skills list
autoApproveThreshold — threshold value for auto-approving tasks
interests (string array, max 20)
topicsToWatch (string array, max 20)
socialHandles (object)
personalContext (string, max 1000 chars)
humorStyle (string)
premiumModel — "grok-4", "gpt-5.4", or "none"
7. Delete (Pause) Agent
DELETE /:id

No body needed. Soft-deletes by setting status to "paused."
Skills Management
8. Enable Skill
POST /:id/skills/:skillId/enable

URL params: id (agent), skillId (skill to enable)
Adds the skill if not already enabled
9. Disable Skill
POST /:id/skills/:skillId/disable

URL params: id (agent), skillId (skill to remove)
Task Queue
10. List Tasks
GET /:id/tasks

Returns the 50 most recent autonomous tasks generated by the agent
11. List Pending Tasks
GET /:id/tasks/pending

Returns only tasks with status "pending" awaiting approval
12. Approve Task
POST /:id/tasks/:taskId/approve

URL param: taskId
Marks a pending task as approved
13. Reject Task
POST /:id/tasks/:taskId/reject

URL param: taskId
Marks a pending task as rejected
Chat & Conversations
14. Chat with Agent
POST /:id/chat

Body:
message (string, required) — max 2000 characters
conversationId (number, optional) — to continue an existing conversation; omit to start a new one
Response: Streamed via Server-Sent Events (SSE)
Chunks: data: {"content": "..."}
Final: data: {"done": true, "conversationId": 123}
Rate limit: Enforces a daily token limit per agent (default 50,000)
15. List Conversations
GET /:id/conversations

Returns all conversation sessions for the agent, newest first
16. Get Messages
GET /:id/messages

Query param: conversationId (optional) — if omitted, returns messages from the most recent conversation
Returns the full message history for that conversation
Soul (Personality)
17. Get Soul Document
GET /:id/soul

Returns the agent's soul/personality definition, when it was last updated, and whether it's still the default template
18. Update Soul Document
PUT /:id/soul

Body: { "soul": "string" } — min 10 chars, max 5000 chars
Overwrites the agent's core personality directive
Knowledge Base
19. Upload Knowledge
POST /:id/knowledge

Body:
type (string, required) — "text" or "url"
content (string, required) — the text itself (max 5000 chars) or a URL to fetch
title (string, optional) — label for the entry
Alternatively: { "text": "..." } or { "url": "..." } shortcuts work too
Limits: Max 20 knowledge entries per agent. URLs are fetched and stripped of HTML. Content is chunked and embedded for semantic search.
20. List Knowledge
GET /:id/knowledge

Returns all uploaded knowledge entries
21. Delete Knowledge Entry
DELETE /:id/knowledge/:knowledgeId

Removes a specific knowledge entry
Memories (Auto-learned)
22. List Memories
GET /:id/memories

Returns all memories (auto-learned facts) grouped by category, with total count
23. Update Memory
PATCH /:id/memories/:memoryId

Body (all optional):
fact (string) — edit the stored fact
pinned (boolean) — pin it so it always surfaces in context
24. Delete Memory
DELETE /:id/memories/:memoryId

Permanently removes a memory
Settings
25. Get Settings
GET /:id/settings

Returns: name, emoji, description, interests, topicsToWatch, socialHandles, enabledSkills, personaTemplate, telegramBotUsername, telegramNotificationLevel, humorStyle, premiumModel
26. Update Settings
PUT /:id/settings

Body (all optional):
name (string, 2-50 chars)
emoji (string, max 10 chars)
description (string, max 500 chars)
interests (array, max 20)
topicsToWatch (array, max 20)
socialHandles (object)
enabledSkills (array of valid skill IDs)
personalContext (string, max 1000 chars)
humorStyle (string)
premiumModel — "grok-4", "gpt-5.4", or "none"
Awareness (Agent Growth)
27. Get Awareness Status
GET /:id/awareness

No parameters
Returns a growth/maturity dashboard:
messageCount, memoriesLearned, conversationCount
phase — "curious", "developing", or "confident"
label — human-readable description of the phase
progress — 0-100 percentage
onChain — booleans for wallet, token, identity status
Telegram Integration
28. Connect Telegram Bot
POST /:id/telegram/connect

Body: { "botToken": "string" } — your Telegram bot token from BotFather
Validates with Telegram, sets up a webhook, stores the connection
29. Disconnect Telegram
DELETE /:id/telegram/disconnect

Removes the webhook and clears all Telegram data
30. Update Telegram Settings
PATCH /:id/telegram/settings

Body: { "notificationLevel": "all" | "important" | "none" }
31. Get Telegram Status
GET /:id/telegram/status

Returns: connected (boolean), botUsername, notificationLevel
That's all 31 Miniclaw endpoints. The available AI models for the premiumModel field are grok-4 (uses xAI's Grok), gpt-5.4 (latest OpenAI), or none (defaults to gpt-5). Let me know if you want me to dive deeper into any specific endpoint or parameter.
